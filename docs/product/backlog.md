# Бэклог

## User story: оставить заявку на консультацию
**Как** посетитель лендинга, который хочет понять, чем ProstoKit поможет в его кейсе,
**я хочу** отправить свои контакты и короткое описание задачи через форму,
**чтобы** получить быстрый ответ от команды и демо подходящих инструментов.

### Acceptance Criteria
- Форма содержит обязательные поля: имя, email и текст сообщения (минимум 10 символов).
- При успешной отправке пользователь видит подтверждение и не теряет введённые данные.
- Форма использует публичный API `/feedback`, который сохраняет заявки в базе и возвращает `201 Created` с данными записи.
- Валидационные ошибки отображаются без перезагрузки страницы.
- Запросы выполняются поверх HTTPS и учитывают CORS для домена фронтенда.

### Заметки по реализации
- Бекенд: FastAPI-эндпойнт с валидацией, записью в БД и возвратом метаданных заявки.
- Фронт: секция лендинга с формой и UI-состояниями загрузки/ошибки/успеха.
- Тесты: интеграционные проверки API (валидный запрос и обработка ошибки валидации).

## User story: подписаться на обновления продукта
**Как** пользователь, который хочет узнавать о новых возможностях ProstoKit заранее,
**я хочу** подписаться на email-рассылку прямо с лендинга,
**чтобы** вовремя получать анонсы свежих инструментов и приглашения в бета-тесты.

### Acceptance Criteria
- Форма содержит обязательный валидируемый email и необязательное имя (минимум 2 символа).
- Повторная подписка с тем же email возвращает понятную ошибку без дублей в базе.
- При успешной подписке пользователь видит подтверждение и остаётся на странице.
- API `/newsletter` создаёт запись в базе и возвращает `201 Created` с данными подписки.

### Заметки по реализации
- Бекенд: FastAPI-эндпойнт с проверкой уникальности email и возвратом конфликта при повторе.
- Фронт: секция лендинга с формой подписки, состояниями загрузки/успеха/ошибки и подсказкой по частоте писем.
- Тесты: позитивный кейс создания подписки и негативные проверки на некорректный email и дублирование.
